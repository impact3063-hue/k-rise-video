## 🎯 分割ルール（優先順位）

### 1️⃣ 最優先: 句読点
```
「、」「。」「！」「？」の直後で必ず分割
例: 「伝説のプロデューサー、」← ここで分割
```

### 2️⃣ 優先: 文節境界
```
名詞・動詞・形容詞などの自立語の前で分割
例: 「あのBTS」/「を」/「日本」/「に」/「導いた」
```

### 3️⃣ 制限: 最大文字数
```
max_chars = 14文字（デフォルト）
推奨範囲: 12-15文字
```

---

## 📝 良い分割例 vs 悪い分割例

### ✅ 良い分割（v3.1）
```
1. "あのBTSを日本に導いた伝説"     (14文字) ← 「伝説」で自然に終了
2. "のプロデューサー、"             (9文字)  ← 句読点で分割
3. "出口氏が、"                     (5文字)  ← 句読点で分割
```

### ❌ 悪い分割（v3.0）
```
1. "あのBTSを日本に導いた伝説の"   (15文字) ← 「の」で中途半端
2. "プロデューサー出口氏が"         (11文字) ← 単語が分断されている
```

## 🐛 トラブルシューティング

### Janomeがない場合
```
⚠️  Warning: Janome not installed. Falling back to simple segmentation.
```
**対処**: `pip install janome`

### 文字数が多すぎる
**原因**: 句読点が少ない台本  
**対処**: 
1. 台本に句読点を追加
2. `max_chars`を小さくする（例: 12）

### 文字数が少なすぎる
**原因**: 句読点が多すぎる台本  
**対処**: 
1. 不要な句読点を削除
2. `max_chars`を大きくする（例: 16）

## 🎓 技術用語

| 用語 | 説明 |
|------|------|
| **形態素解析** | 文章を最小単位（形態素）に分割する技術 |
| **文節** | 意味のまとまりを持つ単語の塊 |
| **自立語** | 単独で意味を持つ単語（名詞、動詞など） |
| **付属語** | 単独では意味を持たない単語（助詞、助動詞など） |
| **Janome** | Pure Pythonの日本語形態素解析ライブラリ |
